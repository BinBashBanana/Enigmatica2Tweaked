<br />
<p>
Fork of Enigmatica 2 (1.12.2)
<br />
<b>This pack is unofficial and is NOT affiliated with the creators of Enigmatica. I have <a href="https://github.com/EnigmaticaModpacks/Enigmatica2/issues/748">permission</a> from NillerMedDild to upload this pack.</b>
</p>
<br />
<h3>Description</h3>
<br />
<p>
This pack is essentially a mashup of Enigmatica 2 normal mode, Direwolf20 1.12.2, and ATM3R (plus a few other mods).
A great all-purpose pack for beginners and experienced players.
It is kitchen-sink style, however it includes Enigmatica 2's quest book to help you along.
This pack is medium-heavy, requiring about 6 GB of ram allocated to perform well.
I'd say the pack is just lighter than ATM3R.
As of version 1.02, some mods from E2Eu are included.
</p>
<p>
This pack does NOT include: refined storage, gregtech, buildcraft, projectE, avaritia, mystical agriculture, others...
</p>

<h3>Important!! Read this!!</h3>
<p>
I highly recommend using a custom build of NormalASM plus OptiFine with this modpack.
You can play without either, but performance will suffer.
I have confirmed that OptiFine is compatible, though some may want to use only NormalASM and not OptiFine.
If you choose to do this, be sure to set `B:onDemandAnimatedTextures=false` back to `true` in `config/normalasm.cfg`.
</p>
<li><a href="https://github.com/BinBashBanana/Enigmatica2Tweaked/releases/download/normalasm-5.14-spongeforge-compat/normalasm-5.14-spongeforge-compat.jar">NormalASM custom build</a></li>
<li><a href="https://optifine.net/adloadx?f=OptiFine_1.12.2_HD_U_G5.jar">OptiFine</a></li>
<p>
For more information about the custom build of NormalASM, see <a href="https://github.com/mirrorcult/NormalASM/pull/1">here</a> and <a href="https://github.com/LoliKingdom/LoliASM/pull/199">here</a>.
</p>

<h4>Recommended JVM arguments</h4>
From <a href="https://www.reddit.com/r/feedthebeast/comments/5jhuk9/modded_mc_and_memory_usage_a_history_with_a/">this reddit post</a>:
<pre class="codeStyle" style="font-family:monospace;background-color:#e0e0e0;padding: 5px 8px;border-radius:4px;overflow:auto">
-Xmx6656M -Xms4G -XX:+UseG1GC -Dsun.rmi.dgc.server.gcInterval=2147483646 -XX:+UnlockExperimentalVMOptions -XX:G1NewSizePercent=20 -XX:G1ReservePercent=20 -XX:MaxGCPauseMillis=50 -XX:G1HeapRegionSize=32M
</pre>
You'll also need Java 8 exactly, and <a href="https://adoptium.net/temurin/releases/?version=8">adoptium.net</a> is the best place to get it IMO.
<br />
<br />
For the server, from <a href="https://mcflags.emc.gs">mcflags.emc.gs</a>:
<pre class="codeStyle" style="font-family:monospace;background-color:#e0e0e0;padding: 5px 8px;border-radius:4px;overflow:auto">
java -Xmx6G -Xms5G -XX:+UseG1GC -XX:+ParallelRefProcEnabled -XX:MaxGCPauseMillis=200 -XX:+UnlockExperimentalVMOptions -XX:+DisableExplicitGC -XX:+AlwaysPreTouch -XX:G1NewSizePercent=30 -XX:G1MaxNewSizePercent=40 -XX:G1HeapRegionSize=8M -XX:G1ReservePercent=20 -XX:G1HeapWastePercent=5 -XX:G1MixedGCCountTarget=4 -XX:InitiatingHeapOccupancyPercent=15 -XX:G1MixedGCLiveThresholdPercent=90 -XX:G1RSetUpdatingPauseTimePercent=5 -XX:SurvivorRatio=32 -XX:+PerfDisableSharedMem -XX:MaxTenuringThreshold=1 -Dusing.aikars.flags=https://mcflags.emc.gs -Daikars.new.flags=true -jar forge-1.12.2-14.23.5.2859.jar nogui
</pre>
Note: you can usually get away with 4 GB, but 6 GB is better especially for pregen.

<h4>Extra notes for servers</h4>
<li>Do not use chunk pregenerator due to incompatibilities with reid/jeid. Use `/forge generate` until it is fixed. See [RoughlyEnoughIDs#14](https://github.com/TerraFirmaCraft-The-Final-Frontier/RoughlyEnoughIDs/issues/14) for more information.</li>
<li>If you want recurrent complex structures in your world, add the mod before you pregenerate the world, then remove it.</li>
<li>If you want LagGoggles on your spongeforge server, you'll need the latest version that doesn't require TickCentral. Use `LagGoggles-THIN-1.12.2-4.11-92.jar`. Clients can still connect with the latest version of LagGoggles.</li>
<li>I have included a patched build of NormalASM that fixes a crash with spongeforge. See issue 194 for more information. You will also need to set `threadPriorityFix=false` in normalasm.cfg (server only).</li>
